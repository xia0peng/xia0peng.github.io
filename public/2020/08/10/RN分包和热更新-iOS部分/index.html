RN分包、增量更新 - iOS部分

Aug 10, 2020 | 性能优化 | 阅读

背景

项目初期所有 React Native 业务都打成一个bundle，客户端在集成JS代码时由于压力过大，导致白屏和内存问题。

方案

将重复的 React Native 代码和第三方库打包成一个基础包，然后各个业务在基础包的 js 环境下运行，这样做可以减少内存开销且热更新的时候只需要下载对应的业务包，减少包大小。

客户端加载方案

基础包预加载，监听基础包加载成功的消息，基础包加载后再加载业务包，顺序和时机控制
